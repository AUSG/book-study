# 14장 보안 HTTP

- **HTTPS (SSL/TLS)**:
  - HTTP에 보안 계층을 더해, 요청과 응답을 네트워크를 통해 전송하기 전에 암호화
  - HTTPS는 기본적으로 SSL/TLS 프로토콜을 사용해 데이터를 안전하게 전송하며, 서버와 클라이언트 간의 핸드셰이크 과정이 필요
- **디지털 암호학**:

  - **대칭키 암호법**: 하나의 키를 공유하여 암호화와 복호화를 모두 수행하며, 키 관리가 어려움
  - **공개키 암호법**: 두 개의 비대칭 키(공개 키와 비공개 키)를 사용하여 데이터를 안전하게 전송(RSA)

- **디지털 서명**:

  - 데이터가 변조되지 않았음을 확인하고, 데이터 작성자의 신원을 확인
  - 비대칭 키를 사용하여 메시지의 무결성을 보장하며, 메시지의 작성자를 인증

- **디지털 인증서**:

  - 인증 기관(CA)에 의해 발급된 신뢰할 수 있는 정보로, 서버 인증서를 통해 서버의 신원을 클라이언트가 검증

- **HTTPS와 SSL 핸드셰이크**:

  - SSL 핸드셰이크 과정을 통해 클라이언트와 서버 간에 보안 채널을 설정하며, 암호화된 데이터 전송
  - SSL 인증서를 통해 사이트의 신뢰성을 검증

- **프록시와 HTTPS 터널링**:
  - HTTPS 트래픽은 프록시 서버가 직접 읽지 못하도록 하며, CONNECT 메서드를 사용하여 특정 호스트와 포트로의 연결을 프록시에게 요청하고, 터널링 방식을 사용해 데이터를 전송
