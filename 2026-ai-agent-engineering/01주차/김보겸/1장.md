# 1장 - 에이전트

## AI Agent의 정의
- 자율 에이전트란 데이터를 스스로 분석하고 환경을 해석하며 컨텍스트에 기반한 결정을 내리도록 설계된 지능형 시스템
- 파운데이션 모델과 강화학습의 발전으로 자율 에이전트의 진화가 가능했음

## 사전 학습이 일으킨 혁명
파운데이션 모델은 자연어 이해와 생성에 강점을 보임
- 자연어 이해: 사용자의 입력을 해석하고 응답
- 컨텍스트 인지 상호작용: 긴 대화에서도 관련 컨텍스트를 유지해 정확도 상승
- 구조화된 생성: 텍스트, 코드, 구조화된 결과 생성으로 분석, 창작 작업 지원

정의한 범위 내에서 의사결정을 내리고, 새로운 정보에 반영해 적응 & 도구 호출 가능
오케스트레이션 프레임워크와 결합시 외부 시스템과 직접 상호작용 후 작업 실행 가능

## 비동기로의 전환
자율 에이전트는 비동기식을 전제로 설계
동시에 여러 작업을 관리하고 새로운 정보에 빠르게 적응, 조건에 따라 우선순위 동적 조정

## 워크플로와 에이전트
모든 작업에 대해 에이전트가 적합한 것은 아님
- 절차적 코드: 입출력이 완전히 고정적인 경우
- 결정적 또는 반자동 워크플로: 로직을 유한한 단께나 분기 표현, 인간 개입이나 추가 에러 처리 지점을 명확히 아는 경우
- 챗봇, RAG: 자연어 이해와 문서 검색 기능 등이 필요하지만 자율적이고 다단계의 계획 수립까지는 필요 없는 경우

| 특성 | 코드 | 워크플로 | 자율 에이전트 |
| :--- | :--- | :--- | :--- |
| 입력 구조 | 완전 예측 가능 스키마 | 유한 분기로 대체로 예측 가능 | 고도로 비정형/새로운 입력 |
| 설명 가능성 | 완전 투명, 감사 용이 | 분기별 감사 추적 명시 | 블랙박스 요소 |
| 지연 | 초저지연 | 중간 지연 | 더 높은 지연 |
| 적응, 학습 | 없음 | 제한적 | 높음(피드백 학습) |
