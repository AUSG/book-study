# 강화학습, Thinking 모델, AI Agent의 관계

LLM의 post-training이 어떻게 추론 능력과 에이전트적 행동을 가능하게 할까?

## 1. 큰 그림으로 LLM 시스템 이해하기
최신 LLM 시스템을 3층 구조로 이해하면 가장 직관적
1) Pre-training: 방대한 데이터를 바탕으로 다음 토큰 예측을 학습
2) Post-training: 사전학습 모델에 추가 학습을 얹어 실제 사용에 적합하게 만듦
3) Agentic Scaffolding: 모델 위에 행동 가능한 시스템 레이어를 얹은 것(모델 자체보다는 시스템 설계에 가까움)

- Pre-training만으로 LLM은 강력한 언어 모델이 되지만, 실제 사용에 필요한 행동 패턴은 충분히 형성되지 않음
    - 인터넷 데이터는 '정답처럼 보이는 문장'은 많지만 '좋은 행동 과정'은 적음
    - 정답의 표면은 학습 가능하지만, 정답에 이르는 안정적인 행동 습관을 충분히 배우기 어려움
    - 이는 next token prediction의 목표 자체가 '그럴듯한 문장'을 생성하는 것이기 때문
- 리서쳐들이 모델이 오래 생각하고 길게 생각할수록 더 좋은 성능을 내는 것을 발견
    - 좋은 모델은 다음과 같은 행동을 보임: 문제를 쪼개고, 가설을 세우고, 중간 결과를 점검하고, 다른 풀이 전략을 시도하고, 계산이나 검색 도구를 사용하고, 자기 실수를 되짚는다

## 2. Post Training의 발전
- 기존 SFT(Supervised Fine-Tuning)은 안정적인 instruction-following을 만드는 데 효과적
- 하지만 SFT만으로는 모델이 '스스로 생각하고 검증하는 행동'을 충분히 학습하기 어려움
- 이를 해결하기 위해 등장한 것이 강화학습 기반 방법론들
    - RLHF(Reinforcement Learning from Human Feedback)
    - RLAIF(Reinforcement Learning from AI Feedback)
- 이 방법론들은 '어떤 행동이 좋은 행동인지'를 보상(reward)으로 학습해 모델의 행동 정책을 개선
    - SFT: 정답 데모를 따라 하게 함
    - RL: 실제로 높은 점수를 받는 방향으로 정책을 조정함
- LLM의 답변 생성 과정을 긴 행동의 시퀀스로 봤을 때, RL은 이런 행동의 시퀀스 속에서 좋은 행동을 하도록 유도
- 특히 DeepSeek-R1, o1과 같은 모델들은 강화학습을 통해 '더 오래 생각하고 길게 생각하는' 능력을 강화

## 3. Thinking 모델
- Thinking 모델은 단순히 답을 길게 쓰는 모델이라고 생각하는 것은 오해. 핵심은 '더 나은 추론 정책'
    - 단순히 긴 답을 출력하는 것이 아니라, 보상을 높이는 방향으로 더 적절한 추론 경로를 택하도록 훈련된 모델
    - OpenAI의 o1 설명
        - 실수를 인식하고 수정
        - 어려운 문제를 작은 단계로 쪼갬
        - 여러 전략을 시도
        - 더 오래 생각하는 것이 유리한 문제에서 실제로 더 많은 test-time compute를 사용
- Test-time compute(실제 답을 생성하는 순간에 추가로 쓰는 계산 자원)를 유연하게 사용
    - 모델이 답을 바로 끝내지 않고 더 많은 중간 토큰을 써서 추론하게 하는 방식
    - 여러 후보를 생성 후 고르기
    - 외부 도구를 써서 답을 보강하기
- RL은 LLM이 어려운 문제에 대해 짧고 즉흥적인 답보다는, 중간 추론과 자기 점검을 거친 답이 더 높은 보상을 받는다는 것을 학습하게 함
    - 즉, RL은 모델이 '더 오래 생각하고 길게 생각하는' 능력을 강화
        - 즉답보다 중간 추론이 유리할 수 있음
        - 첫 시도보다 재검토가 유리할 수 있음
        - 계산이나 검산이 유리할 수 있음
        - 한 전략이 막히면 다른 전략이 유리할 수 있음
    - 단순한 문장 생성을 넘어, 복잡한 문제를 해결하기 위한 추론 과정을 학습하게 함

- 심화: DeepSeek-R1의 학습 기법(SFT+RL)
    - DeepSeek-R1의 파이프라인은 대략 이런 철학으로 이해할 수 있다.
    1. RL로 reasoning 행동을 끌어낸다.
    2. 그중 좋은 결과를 샘플링해 데이터로 모은다.
    3. 다시 SFT로 안정화한다.
    4. reasoning/general capability를 대상으로 다시 RL한다.
    - SFT와 RL을 번갈아 조합해 유능함과 사용성을 함께 만든다.
    - DeepSeek-R1 계열은 다음 메시지를 준다.
        - 프리트레이닝이 잠재력을 만든다.
        - RL이 reasoning 행동을 끌어낸다.
        - SFT가 그 행동을 사용자 친화적으로 정리한다.
        - 다시 RL이 성능을 밀어올린다.



## 4. AI Agent란 무엇인가
- AI Agent는 모델 자체가 아니라, 모델을 중심으로 한 시스템. 모델 위에 행동 가능한 시스템 레이어를 얹은 것
- 주요 구성 요소:
    - 도구(tool) 사용
    - 반복 루프(loop)
    - 메모리(memory)
    - 계획(planning)
    - 상태 관리(state management)
    - 가드레일(guardrails)
- 에이전트의 중요 능력 중 하나는 행동 선택
    - 지금 당장 답해도 될지, 어떤 도구를 써야 할지, 사용자의 피드백을 받아야 할지 등
    - 이런 것은 전형적인 policy 문제. RL은 바로 이런 정책 학습과 잘 맞는다.
- 좋은 Thinking 모델은 좋은 Agent의 두뇌가 될 수 있음
