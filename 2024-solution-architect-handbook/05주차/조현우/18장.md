# 솔루션 아키텍처 문서

정의: SAD(Solution Architecture Document)


### SAD의 목적
- 모든 이해관계자에게 종단 간 애플리케이션 솔루션 전달
- 애플리케이션 설계에 대한 높은 수준의 아키텍처와 다양한 보기를 제공
- 기능적 및 비기능적 요구 사항을 충족하는 방법을 탐색

### SAD의 관점
자신의 상황을 파악하고 이해관계자의 관점에서 문제를 보는 것이 좋다.
아래와 같은 관점으로 바라볼 수 있다.
- 비즈니스 관점
- 논리적 관점
- 프로세스 관점
- 배포 관점
- 구현 관점
- 데이터 관점
- 운영 관점


### SAD의 구조
테크스펙과 비슷한 것 같다.
